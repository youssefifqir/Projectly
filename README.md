# Projectly

Generated Spring Boot backend application with RESTful APIs.

## 🚀 Quick Start

### Prerequisites
- Java 17 or later
- Maven 3.8+
- PostgreSQL 15+ (for production)
- Docker & Docker Compose (optional)

### Running Locally

#### Development Mode (H2 in-memory database)
```bash
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

#### Test Mode
```bash
mvn test
```

#### Production Mode (PostgreSQL required)
```bash
# Update application-prod.yml with your database credentials
mvn spring-boot:run -Dspring-boot.run.profiles=prod
```

### Building the Application
```bash
mvn clean package
java -jar target/Projectly-0.0.1-SNAPSHOT.jar
```

## 🐳 Docker Deployment

### Using Docker Compose
```bash
docker-compose up --build
```

This will start:
- Application on port 8080
- PostgreSQL database on port 5432

### Building Docker Image
```bash
docker build -t projectly:latest .
```

## ⚙️ Configuration

### Server URL

`app.server.url` must be set to the public-facing URL of your application. It is used by:
- Swagger/OpenAPI for server URL in generated docs
- OAuth redirect URIs (if Google auth is enabled)
- Password reset / email verification links (if email features are enabled)

Set via environment variable:
```bash
export SERVER_URL=https://your-domain.com
```

Or in `application.yml`:
```yaml
app:
  server:
    url: https://your-domain.com
```

### Database Configuration

#### Development (H2)
Automatic in-memory database - no configuration needed

#### Production (PostgreSQL)
Update `application-prod.yml`:
```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/Projectly_prod
    username: your_username
    password: your_password
```

Or use environment variables:
```bash
export DB_PASSWORD=your_secure_password
```
